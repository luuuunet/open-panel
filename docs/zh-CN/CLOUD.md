# 云厂商整合指南

> 入口：**自动化 → 自动化运维 → 云厂商整合**

OWPanel 将阿里云、腾讯云、AWS、Google Cloud、IBM 等常见云能力整合到**同一面板**，无需在多个云控制台之间切换。

---

## 1. 能力对照

| 云厂商产品 | OWPanel 模块 | 说明 |
|-----------|-------------|------|
| 阿里云 OSS / 腾讯 COS / AWS S3 / GCS | **对象存储** | 本机目录、MinIO、多云 S3 同步与迁移 |
| 阿里云 DNS / DNSPod / Cloudflare | **DNS 管理** | 自动检测解析、一键修复 A 记录 |
| 云快照 / 自动备份 | **备份** | 定时打包网站/数据库，可上传 OSS |
| GCP Uptime Check | **可用性监控** | 外网 HTTP 探测，从网站一键导入 |
| 云监控告警 | **自动化运维** | CPU/内存/磁盘阈值 + Webhook |
| 负载均衡 SLB/ALB | **服务器集群** | 多节点与 MySQL 主从（可视化配置） |
| 阿里云 DM / Amazon SES | **邮件** | 第三方发信 API |

---

## 2. 推荐流程（从云控制台迁移）

### 第一步：接入对象存储

1. 打开 **文件 → 对象存储**
2. 点击 **添加存储**，选择云厂商（阿里云/AWS 等）
3. 填写 Endpoint、Bucket、AccessKey/SecretKey
4. **连接测试** 通过后保存

> 从「云厂商整合」页点击 **去添加** 可自动预选云厂商类型。

### 第二步：一键迁移包

1. 打开 **自动化运维 → 云厂商整合**
2. 在对应云厂商卡片选择已添加的 **OSS 存储**
3. 点击 **一键开启 xxx 能力包**

自动完成：

- 服务监控 + 自动重启 + SSL 续期
- 全站可用性探测（5 分钟）
- 每日网站/数据库备份（可选上传 OSS）
- 资源告警 + 内存自动释放
- OSS wwwroot 每日同步任务（若选了存储）

### 第三步：补全 DNS 与 Webhook

- **DNS**：域名 → DNS 管理 → 接入服务商 → 域名检测 → 一键修复
- **Webhook**：自动化运维 → 策略设置 → 填写钉钉/飞书/企业微信机器人地址

---

## 3. 网站页快捷操作

在 **网站管理** 工具栏：

- **全部加监控** / **全部加备份** — 批量自动化
- **OSS 下拉框** — 备份任务创建时指定上传到哪个云存储
- **云厂商整合** — 跳转到整合中心

---

## 4. 备份 + OSS

**备份页** 快捷模板上方可选择 **默认 OSS 上传**，一键创建的任务会自动关联该存储。

已有备份任务可在编辑时指定 `oss_storage_id`，或在云迁移包中 **关联 OSS**。

---

## 5. OSS 同步与跨云迁移

**对象存储 → 同步/迁移任务** 支持：

| 模式 | 用途 |
|------|------|
| 上传到云端 | 本机 → 阿里云/AWS 等 |
| 从云端下载 | 云 → 本机 |
| 双向同步 | 保持两端一致 |
| 跨存储迁移 | 阿里云 → AWS 等 |

可配置 Cron 定时执行，适合替代云厂商「定时快照到对象存储」类需求。

---

## 6. API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/cloud/hub` | 整合状态与各云厂商配置概览 |
| POST | `/api/cloud/presets/:vendor` | 应用迁移包（vendor: aliyun/tencent/aws/gcp/ibm） |

请求体示例：

```json
{
  "oss_storage_id": 1,
  "include_ops": true,
  "link_backup": true,
  "create_sync": true
}
```
